RDTP (Reliable Data Transfer Protocol) — системы надежной передачи данных, построенной поверх ненадежного протокола UDP.

Передать файл от Отправителя (Sender) к Получателю (Receiver) без потерь и искажений, несмотря на то, что сеть (эмулируемая программой) постоянно теряет пакеты, задерживает их и путает порядок доставки.

Алгоритм надежности: Go-Back-N (Возврат на N).
Отправитель может отправить сразу несколько пакетов (размер окна), не дожидаясь подтверждения каждого.
Если происходит потеря пакета, отправитель по таймауту повторно высылает все пакеты, начиная с потерянного.

Симуляция плохой сети: В класс сокета встроена искусственная "поломка":
С вероятностью 20% пакет просто удаляется (не отправляется).
Добавляется случайная задержка (до 50 мс).

Отправитель (RdtSender)
1. Читает файл и режет его на кусочки по 1024 байта.
2. К каждому кусочку добавляет заголовок:
3. SeqNum: Порядковый номер, чтобы собрать файл в правильном порядке.
4. Checksum: проверка целостности.
5. Отправляет сразу пачку пакетов, не ожидая ответа. Запускает Таймер.
6. Таймер истек, а подтверждение (ACK) не пришло — значит, пакеты потерялись. Отправитель "возвращается назад" (Go-Back) и шлет всё окно заново.

Получатель (RdtReceiver)
1. Ждет пакет с конкретным номером (expectedSeqNum). Сначала ждет 0.
2. Проверка:
Если пришел пакет №0 (как ожидалось) -> Пишет данные в файл, увеличивает ожидание на 1, шлет ACK 0.
Если пришел пакет №2 (а ждали 0) -> Это пакет "вне очереди". В алгоритме Go-Back-N он игнорируется. Получатель повторно шлет ACK последнего успешного пакета (или молчит, заставляя отправителя уйти в таймаут).
3. При получении флага FIN закрывает файл.
